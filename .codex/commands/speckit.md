## Outline

If the user wants a specific Speckit step, direct them to the matching command:

- `/speckit.specify`
- `/speckit.clarify`
- `/speckit.plan`
- `/speckit.tasks`
- `/speckit.analyze`
- `/speckit.checklist`
- `/speckit.implement`
- `/speckit.constitution`
- `/speckit.taskstoissues`

If no step is specified, explain the workflow order:

1. `/speckit.specify`
2. `/speckit.clarify`
3. `/speckit.plan`
4. `/speckit.tasks`
5. `/speckit.analyze`
6. `/speckit.implement`

Keep the response short and plain. Tell the user which command to run next.
