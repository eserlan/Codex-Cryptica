# Implementation Plan: Directional Vault Synchronization

**Branch**: `093-directional-vault-sync` | **Date**: 2026-04-25 | **Spec**: [specs/093-directional-vault-sync/spec.md](spec.md)
**Input**: Feature specification from `/specs/093-directional-vault-sync/spec.md`

## Summary

Refactor the vault synchronization system from a bidirectional merge model to an explicit unidirectional "Push/Pull" (Save/Load) model. Establish the browser's internal storage (OPFS/IDB) as the canonical master and the local filesystem as a secondary mirror. This improves UX clarity, data safety (via Safety Gates), and performance (by removing FS sync from the vault switch flow).

## Technical Context

**Language/Version**: TypeScript 5.9.3, Svelte 5 (Runes)
**Primary Dependencies**: SvelteKit, `@codex/sync-engine`, `@codex/vault-engine`, `idb`
**Storage**: OPFS (Primary), IndexedDB (Metadata/Cache), Local Filesystem (Mirror via File System Access API)
**Testing**: Vitest (Unit/Integration), Playwright (E2E)
**Target Platform**: Modern Web Browsers (supporting File System Access API)
**Project Type**: Web Application (Monorepo with specialized logic packages)
**Performance Goals**: Near-instant vault switching (<500ms for metadata load), efficient incremental Push/Pull operations.
**Constraints**: Browser security sandboxing, fragile/revocable filesystem permissions, large vault handling (5000+ entities).
**Scale/Scope**: Impacts core data flow across 3 workspace packages and the main UI layer.
