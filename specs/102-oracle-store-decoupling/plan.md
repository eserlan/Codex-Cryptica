# Implementation Plan: Oracle Store Decomposition

**Branch**: `102-oracle-store-decoupling` | **Date**: 2026-05-19 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/102-oracle-store-decoupling/spec.md`

## Summary

Decompose the monolithic `oracle.svelte.ts` (902 lines, #1 God-file) into six specialized managers (UI, Context, Chat, Actions, Settings, and Reconciliation) located in a new `oracle/` subdirectory. The main `OracleStore` will be refactored into a thin facade to maintain backward compatibility for 48+ existing importers while enforcing constructor-based dependency injection for all internal managers.

## Technical Context

**Language/Version**: TypeScript 6.0.3, Svelte 5 Runes  
**Primary Dependencies**: `@google/generative-ai`, `idb` (IndexedDB), `flexsearch`, `comlink`  
**Storage**: OPFS (Vault Files), IndexedDB (Chat History, Sync Registry), LocalStorage (UI State)  
**Testing**: Vitest  
**Target Platform**: Web (Browser-native APIs)  
**Project Type**: Web Application Store Refactor  
**Performance Goals**: Sub-100ms UI responsiveness; No blocking of main thread during context assembly  
**Constraints**: MUST maintain backward compatibility for 48+ importers; MUST use constructor-based DI; MUST preserve Comlink proxy behavior for Worker communication.  
**Scale/Scope**: Refactoring ~900 lines of state/logic; affecting 48+ files.

## Constitution Check

_GATE: Must pass before Phase 0 research. Re-check after Phase 1 design._

1. **Library-First**: The logic is tied to the web app's reactive state, but the `oracle-engine` package is already isolated. This refactor improves the "thin UI layer" requirement. (PASS)
2. **TDD**: Requirement SC-002 and SC-004 mandate 100% regression testing and 80% new coverage. (PASS)
3. **Simplicity & YAGNI**: Decomposing a God-file is a direct response to a known complexity risk. (PASS)
4. **AI-First Extraction**: Enhances the Oracle's internal structure. (PASS)
5. **Privacy**: Maintains client-side processing (OPFS/IndexedDB). (PASS)
6. **Clean Implementation**: Uses Svelte 5 Runes and proper type safety. (PASS)
7. **User Documentation**: Does not change user-facing behavior, but should verify if any internal documentation needs updates. (PASS)
8. **Dependency Injection**: Explicitly mandated by FR-006. (PASS)
9. **Natural Language**: N/A for internal refactor. (PASS)
10. **Quality & Coverage**: Targets 80% coverage for new managers (exceeding store goal of 50%). (PASS)

**Post-Design Re-evaluation (2026-05-19)**: The proposed manager-based architecture with constructor-based dependency injection (as detailed in `contracts/managers.md`) strictly adheres to ADR 007 and Constitution Article VIII. The design ensures that each logic domain is isolated and testable without circular dependencies, fulfilling the "Simplicity" and "TDD" mandates. (CONFIRMED)
